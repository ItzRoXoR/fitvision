# Приложение для мониторинга активности

Мобильное приложение для отслеживания физической активности: шаги, тренировки, калории, дистанция. Пользователь регистрируется, заполняет профиль, выбирает тренировки из каталога, проходит их с таймером и видит свою статистику по целям. Экран «Прогресс» отображает историю активности в виде графиков за выбранный период. Встроенный модуль FitVision позволяет загрузить своё фото и получить AI-прогноз внешнего вида через 1, 3 или 6 месяцев на основе накопленной статистики активности.

## Архитектура

Проект состоит из четырёх частей.

**Бэкенд** (`backend/`) — REST API на Node.js с фреймворком NestJS. Принимает запросы от мобильного клиента, проверяет данные через class-validator, аутентифицирует через JWT и работает с базой данных PostgreSQL.

**ML-бэкенд** (`ml-backend/`) — Python-сервис на FastAPI. Принимает фото пользователя и параметры активности, вычисляет Activity Score, формирует промпт и запускает Stable Diffusion v1-5 (img2img) для генерации прогноза внешнего вида. Работает на локальном GPU.

**SDK** (`fitness-sdk/`) — отдельная Android-библиотека на Kotlin, которую подключает мобильное приложение. Содержит все репозитории и сервисы: работает с бэкендом через Retrofit и OkHttp, считает шаги через Sensor API, отправляет запросы на генерацию в ML-бэкенд.

**Мобильное приложение** (`mobile/`) — Android-приложение на Kotlin. Слой представления построен на Jetpack Compose по паттерну MVVM: экраны подписываются на состояние из ViewModel, ViewModel обращается к SDK через интерфейсы.

## Стек технологий

| Часть | Технологии |

| Бэкенд | Node.js, NestJS, PostgreSQL, JWT, bcrypt |
| ML-бэкенд | Python, FastAPI, Stable Diffusion v1-5, diffusers, PyTorch (CUDA) |

## Функциональность

- **Регистрация и профиль** — пол, дата рождения, вес, рост, дневные цели по шагам и калориям; все поля проверяются на допустимые диапазоны значений
- **Главный экран** — шаги, сожжённые калории, дистанция, ИМТ за текущий день
- **Ручной ввод активности** — кнопка «+» на главном экране открывает диалог для ввода шагов и сожжённых калорий вручную (суммируется к уже накопленным за день)
- **Шагомер** — автоматический подсчёт шагов через Sensor API; отключён по умолчанию, включается переключателем в профиле
- **Тренировки** — каталог с фильтрацией по типу, сложности, длительности и группе мышц; избранное; сессии с таймером и подсчётом калорий
- **Прогресс** — история активности с выбором показателя (шаги / калории) и периода (3 дня / неделя / месяц); данные отображаются в виде столбчатого графика с итоговой сводкой: среднее за день, общая сумма, количество активных дней
- **FitVision** — AI-прогноз внешнего вида: пользователь выбирает фото из галереи, указывает тип активности, количество дней в неделю, период (1/3/6 месяцев) и цель; в расчёт берётся средняя активность за последние 30 дней из истории; модель возвращает изображение «после» с цветовой индикацией режима

### Режимы FitVision

| Activity Score | Режим | Цвет рамки | Описание |
|---|---|---|---|
| < 15 | Набор веса | Красный | Последствия низкой активности |
| 15–40 | Потенциал | Оранжевый | Мотивирующий прогноз при начале тренировок |
| ≥ 40 | Реальный прогресс | Зелёный | Результат текущего уровня активности |

## Запуск

Для работы требуются три отдельных процесса.

### 1. База данных и NestJS-бэкенд

Из директории `backend/`:

```bash
docker-compose up postgres -d   # запустить PostgreSQL в фоне
npm install                     # первый раз
npm run setup                   # первый раз: миграции + сид каталога тренировок
npm run seed:demo               # опционально: демо-пользователь с историей активности за месяц
npm run dev                     # сервер на порту 3000
```

Начиная со второго запуска достаточно:

```bash
docker-compose up postgres -d
npm run dev
```

Проверить работоспособность: `http://localhost:3000/api/health` — должен вернуть `200 OK`.

#### Демо-пользователь

`npm run seed:demo` создаёт учётную запись с 30 днями истории активности, в которой цели по шагам и калориям стабильно перевыполняются:

| Поле | Значение |
|---|---|
| Логин | `aleksey` |
| Пароль | `demo1234` |
| Профиль | Алексей Горский, муж., 1996-03-15, 82 кг, 181 см |
| Цель по шагам | 10 000 / день |
| Цель по калориям | 500 ккал / день |

### 2. ML-бэкенд

Требования: Python 3.11+, PyTorch с поддержкой CUDA, видеокарта с ≥ 6 ГБ VRAM.

Из директории `ml-backend/`:

```bash
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124   # первый раз, CUDA 12.x
pip install -r requirements.txt  # первый раз
uvicorn main:app --host 0.0.0.0 --port 8000
```

При первом запуске скачивается модель Stable Diffusion v1-5 (~4 ГБ). Последующие запуски занимают ~30 секунд. Готовность обозначается строкой `Model ready`.

### 3. Мобильное приложение

1. Открыть папку `mobile/` в Android Studio
2. Дождаться синхронизации Gradle
3. Создать AVD (рекомендуется Pixel 6, API 33)
4. Запустить на эмуляторе

Эмулятор обращается к хост-машине через `10.0.2.2`, поэтому оба бэкенда должны быть запущены до старта приложения.

### Порядок запуска

```
PostgreSQL (Docker) → NestJS (npm run dev) → ML-бэкенд (uvicorn) → Android-эмулятор
```

## Сборка APK для реального устройства

При запуске на реальном телефоне адрес `10.0.2.2` недоступен — нужно указать адреса, достижимые из сети устройства. URL-адреса бэкендов задаются в `mobile/local.properties` и вшиваются в APK через `BuildConfig` при сборке.

### Cloudflare Tunnel (любая сеть, в том числе мобильный интернет)

Запустите два туннеля — по одному на каждый бэкенд:

```bash
# терминал 1
cloudflared tunnel --url http://localhost:3000

# терминал 2
cloudflared tunnel --url http://localhost:8000
```

Каждая команда выводит публичный HTTPS-адрес вида `https://xxxx.trycloudflare.com`. Подставьте их в `mobile/local.properties`:

```properties
backend.url=https://xxxx.trycloudflare.com
ml.backend.url=https://yyyy.trycloudflare.com
```
Адреса меняются при каждом перезапуске `cloudflared`.

|---|---|---|
| Вес | 20 кг | 300 кг |
| Рост | 100 см | 250 см |
| Цель по шагам | 500 | 50 000 |
| Цель по калориям | 50 ккал | 5 000 ккал |
| Шаги (ручной ввод) | 0 | 50 000 |
| Калории (ручной ввод) | 0 | 5 000 ккал |
